Architectural Approach

How should we structure the code for future "Skill Expansion"?

A

Scene-Centric (Monolithic Scene)

All logic stays inside the Phaser Scene. Easier to follow but gets messy as you add features.

B

Component-Driven (Recommended)

Separate classes for Card, Grid, and ScoreManager. Clean, testable, and easy to expand.

Scene-Centric Structure
  • ✓ Fast to implement
  • ✓ Simple data flow
  • ✗ Hard to reuse components
  • ✗ Large files (500+ lines)
Component-Driven Structure
  • ✓ Senior Engineering Standard
  • ✓ Unit testable components
  • ✓ Easy to add new game modes
  • ✗ Slightly more boilerplate
